## 4. GitHub release (markdown)

Title: **v1.0.2 — Game Pass support + missing-UI and localization fixes**

### Fixed
- **Game Pass / Microsoft Store launch crash.** BLSE died at startup with `MissingMethodException: ILGenerator.MarkSequencePoint`. ModReady only shipped the net472 `Win64_Shipping_Client` build, and Game Pass runs the .NET (Core) runtime where that Harmony/MonoMod path calls a removed API. All four dependency modules now also ship a **.NET 6 `Gaming.Desktop.x64_Shipping_Client`** build. Steam/GOG are byte-for-byte unchanged; the engine picks the right folder per runtime.
- **Missing UI injections on code-generated prefabs.** Bannerlord 1.2+ ships code-generated prefabs (`*.GauntletUI.AutoGenerated.*.dll`) that bypass XML loading, so UIExtenderEx patches editing those prefabs (e.g. the Courier/Messenger "Send Messenger" button on `EncyclopediaHeroPage`) silently never applied. UIExtenderEx now forces the XML load path when a prefab patch is registered, so patches reach generated prefabs too — fixes the whole class, not just Courier.
- **MCM localization.** Option labels now resolve through `TextObject`, so other mods' language packs (Russian, Japanese, Ukrainian, …) display translated instead of English.

**Full Changelog**: https://github.com/Trashpanda62/modready/compare/v1.0.1...v1.0.2
