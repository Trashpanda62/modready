# ModReady changelog

## v1.0.2 — 2026-06-20

**Fix: Game Pass / Microsoft Store crash — BLSE failing at launch with
`MissingMethodException: ILGenerator.MarkSequencePoint`. Adds Game Pass support.**

Steam/GOG run Bannerlord on **.NET Framework** and load each module's
`bin\Win64_Shipping_Client\` (net472). The **Game Pass / Microsoft Store** build
runs Bannerlord on **.NET (Core)** and loads `bin\Gaming.Desktop.x64_Shipping_Client\`
instead. Previous versions shipped only the net472 `Win64_Shipping_Client` folder,
so on Game Pass the loader fell back to the net472 DLLs on the Core runtime —
where Harmony/MonoMod's net472 build calls the Framework-only
`ILGenerator.MarkSequencePoint` (removed on .NET) and throws inside BLSE's
LauncherEx *before any module finishes loading*. All four dependency modules now
ship a second **`bin\Gaming.Desktop.x64_Shipping_Client\`** folder built for
**.NET 6**, like other cross-platform modules. Steam/GOG users are unaffected —
the net472 build is byte-for-byte unchanged; the engine picks the right folder
per runtime.

**Fix: UI injections didn't appear on code-generated prefabs (e.g. the
Courier / Messenger mod's "Send Messenger" button).**

Bannerlord 1.2+ ships code-generated prefabs (`*.GauntletUI.AutoGenerated.*.dll`).
When a prefab has a generated variant, the engine instantiates it from compiled
code and never reads its XML — so UIExtenderEx prefab patches that edit the prefab
XML (e.g. injecting into `EncyclopediaHeroPage`) silently never applied, even
though the mod's ViewModel mixin attached fine. UIExtenderEx now forces movies onto
the XML load path whenever a prefab patch is registered (matching upstream
behavior), so patches reach generated prefabs too. Fixes the whole class of
"button/textbox missing" reports, not just Courier.

**Fix: MCM ignored other mods' localization / language packs.**

MCM stripped the `{=key}` localization tag from option labels and showed the
English fallback, instead of resolving the key through the engine's `TextObject`.
So translations supplied by language packs (Russian, Japanese, Ukrainian, …) —
e.g. for XorberaxLegacy's settings — never appeared. MCM now resolves keyed
strings through `TextObject` first (language pack → translated text, else the
fallback), guarded so any failure or an early call falls back to the old strip
and can never blank or crash the Mod Config panel. English users see no change.

## v1.0.1 — 2026-06-19

**Fix: MCM no longer fails with a "dependency conflict" on startup.**

The dependency modules carried a hard assembly reference to an internal helper
DLL (`BetaDeps.Harmony.dll`) that only shipped with the separate BetaDeps mod.
Since ModReady deliberately does **not** include the BetaDeps module, that
reference had nothing to resolve against, and BLSE's loader rejected MCM with:

> Mod Configuration Menu v5.MCM Submodule could not be loaded correctly due to
> a dependency conflict.

The dependency modules are now fully self-contained — the helper DLL is bundled
with the Harmony module (which loads first), so every reference resolves with no
BetaDeps module present. No load-order or configuration change needed; just
install v1.0.1 over v1.0.0.

## v1.0.0 — 2026-06-19

First standalone release. The four BUTR dependency modules most modern
Bannerlord mods need — Harmony, UIExtenderEx, ButterLib, and MCM — in one
manual-install zip. Requires BLSE (installed separately). Clean-room BetaDeps
builds; credit BUTR.
